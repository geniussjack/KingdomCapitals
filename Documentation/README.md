## YAML Format

The documentation uses YAML (YAML Ain't Markup Language) for:
- **Human-readable**: Easy to read and understand
- **Structured**: Organized in logical sections
- **Machine-parseable**: Can be processed by tools
- **Version-controllable**: Easy to track changes in git

### Viewing YAML Files

**Text Editors:**
- Visual Studio Code (with YAML extension)
- Sublime Text
- Notepad++
- Any text editor

**Online Viewers:**
- [YAML Lint](http://www.yamllint.com/) - Validate and view
- [Online YAML Parser](https://yaml-online-parser.appspot.com/)

**IDE Integration:**
- Most modern IDEs have built-in YAML support
- Syntax highlighting and validation
